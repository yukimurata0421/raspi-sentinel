# セキュリティポリシー

## サポート対象

セキュリティ修正は以下を対象に実施します。  

- `v0.8.x` ラインの最新 stable タグ（現在は `v0.8.0`）
- `v0.9.x` open beta 公開後の beta ライン

古いタグは原則バックポート対象外です。
次のリリースタグまでの暫定修正は `main` を参照してください。

## 脆弱性報告

公開前に、必ず非公開チャネルで報告してください。

推奨窓口:

- GitHub Security Advisories（非公開報告）
- またはメンテナ連絡チャネル

報告時に含める内容:

- 影響バージョン / タグ
- 再現手順
- 影響範囲
- 秘匿情報を除いたログや証跡

公開 Issue には webhook URL や token を貼らないでください。  
非機密の機能不具合は公開 Issue で報告して問題ありません。

## 開示と修正の流れ

1. 受領・再現確認
2. 修正と回帰テスト追加
3. 修正版リリースとリリースノート公開
4. 修正版公開後に詳細開示

応答方針: best effort（SLA なし）。初回トリアージ応答は通常 7 日以内を目安にします。
重大な問題（認証情報漏えい、RCE など）は、可能な範囲で同一週内の修正版公開を優先します。

## 運用上の注意

- `/etc/raspi-sentinel/config.toml` は `root:root` + `0600` を推奨
- command 文字列に長期シークレットを直書きしない
- webhook URL / token をログや events に露出させない
- 本番で timer を有効化する前に `validate-config --strict` を実行
