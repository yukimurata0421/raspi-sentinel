## raspi-sentinel v0.3.1

Maintenance and clarity release: **recovery state is fully modeled with `TargetState`**, CI guards the policy layer, and **`run-once` returns distinct exit codes** for automation.

### Added

- `PROCESS_CHECK_NAMES` in `policy.py` for the “process error” branch (clearer than a long `or` chain).
- Expanded `tests/test_policy.py` (clock freeze confirmed, HTTP probe failed, time sync broken skew, recovered-from-clock-skew, `process_error`).
- CI: **Ruff** (`ruff check`, `ruff format --check`); pytest coverage includes **`policy`**, **`status_events`**, **`time_health`**; separate **coverage gates** for policy+status (≥85%) and checks+recovery (≥88%).

### Changed

- **Recovery** uses `TargetState` end-to-end: `TargetState.from_dict` → mutate → **`merge_into(raw_dict)`** (no mixed dict vs model access).
- **`TargetState`**: `last_records_processed_total`, `records_stalled_cycles`, **`merge_into()`**; progress-check fields round-trip with semantic stats stall logic.
- **`apply_records_progress_check`**: moved from `monitor_stats.py` to **`checks.py`**.
- **`state_helpers`**: `target_state()` lives here; **`runtime_state.py` removed**.
- **`state.py`**: empty-state returns use **`copy.deepcopy(DEFAULT_STATE)`**.
- **`run-once` exit codes**: `0` = all OK, `1` = unhealthy cycle, `2` = reboot requested (no longer folded to `0`). See README.
- README: exit code table; Tests/CI commands aligned with the workflow.

### Removed

- Unused `cli._classify_target_status` / `_classify_target_reason` wrappers (tests call `status_events` directly).

### Upgrade notes

- No config format break from v0.3.0. Replace the package and restart the timer/service as usual.
- If you relied on **`run-once` always exiting 0** on failure, update scripts or systemd units to treat **`1` / `2`** as documented.
