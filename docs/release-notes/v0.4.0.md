# raspi-sentinel v0.4.0

Released: 2026-04-11

## Highlights

- Added operator-facing `validate-config` enhancements and stricter warning visibility.
- Added `run-once --json` reporting for machine-readable one-cycle results.
- Hardened state durability with lock-based lost-update prevention and explicit persistence checks.
- Added state corruption handling:
  - invalid `state.json` is quarantined (`state.json.corrupt.<timestamp>`)
  - cycle enters limited mode and blocks disruptive actions (`restart`/`reboot`) for safety
  - `events.jsonl` records `state_corrupted` / `state_load_error`
- Split runtime orchestration out of CLI into `engine.py` for clearer module responsibilities.
- Added safe-by-default command execution (`shell=False`) with explicit opt-in:
  - `command_use_shell`
  - `dns_check_use_shell`
  - `gateway_check_use_shell`
  - `maintenance_mode_use_shell`
- Added multi-generation events rotation and state size/reboot-history guards.
- Improved Discord webhook retry handling for HTTP `429` `Retry-After`.

## Quality

- CI continues to enforce Ruff + pytest with branch coverage (`--cov-branch`).
- Project coverage gate remains `--cov-fail-under=80`.

## Upgrade notes

- If your config relies on shell syntax (`|`, `&&`, redirections, subshell), set the corresponding `*_use_shell=true`.
- Review `config/raspi-sentinel.example.toml` for updated defaults and new global safety controls.
