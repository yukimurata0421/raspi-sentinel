# raspi-sentinel v0.8.0 (Draft)

Status: draft summary for upcoming `v0.8.x` release work.

## Highlights

- Recovery and policy path cleanup:
  - removed legacy `semantic_clock_*` failure-only reboot branch in `recovery`
  - reboot gating now aligns to policy reason and observation-derived confirmed clock signals
  - removed redundant `safe_int(... ) or default` patterns in network/clock signal builders
- State durability hardening:
  - `_quarantine_corrupt_state` now stops when timestamp candidate slots are exhausted
  - logs explicit error instead of reusing an occupied quarantine path
- Checks package compatibility cleanup:
  - removed underscore compatibility aliases in `checks.__init__`
  - branch tests now import helper functions from owning modules directly
- Additional regression coverage:
  - tiered-store behavior when `require_tmpfs=true` and no durable store
  - corrupt-state quarantine slot exhaustion
  - checks monkeypatch isolation across sequential target runs
- Contract note update:
  - documented timestamp semantics for epoch fields vs host-local ISO rendering

- Verified with:
  - targeted test suite in development tree
  - `validate-config`
  - `run-once --json` (dry-run and non-dry-run)
- Result: successful validation and `overall_status: ok`.
